# 建军照片自动同步系统

## 快速开始

### 1. 文件结构
```
jianjun/auto_sync/
├── monitor.sh              # 监控脚本（检测新照片）
├── process_new_photos.sh   # 处理脚本（处理新照片）
├── album_state.json        # 相册状态记录
├── monitor.log             # 监控日志
└── process.log             # 处理日志
```

### 2. 工作流程

#### 监控阶段
```bash
# 手动执行监控
./jianjun/auto_sync/monitor.sh
```

这个脚本会：
1. 检查照片应用是否打开
2. 截取照片应用界面
3. 保存截图供 AI 分析

#### 处理阶段
```bash
# 处理新照片（需要先通过 AI 识别出照片数量）
./jianjun/auto_sync/process_new_photos.sh <上次数量> <当前数量>
```

这个脚本会：
1. 计算新增照片数量
2. 记录到日志
3. 准备处理流程（需要完善导出功能）

### 3. 状态文件

`album_state.json` 记录：
- `last_photo_count`: 上次检测到的照片数量
- `last_check_time`: 上次检查时间
- `new_photos_since_last`: 自上次以来的新照片列表
- `total_synced`: 总共同步的照片数量

### 4. 当前功能状态

| 功能 | 状态 | 说明 |
|------|------|------|
| 照片应用监控 | ✅ 可用 | 定期截图照片应用 |
| AI 识别照片数量 | ⚠️ 部分可用 | 需要通过 OpenClaw session 调用 |
| 自动导出照片 | 🔍 探索中 | 需要 AppleScript / 快捷指令 |
| 照片内容分析 | ⚠️ 部分可用 | AI 可以分析截图，但需高质量图片 |

### 5. 使用建议

#### 当前可以做的：
1. **定期手动检查**: 每天或每小时执行 `monitor.sh`
2. **截图分析**: 通过截图 + AI 分析了解相册变化
3. **记录状态**: 手动更新 `album_state.json`

#### 需要进一步探索：
1. **自动导出**: 研究 AppleScript 控制照片应用
2. **快捷指令**: 创建导出建军相册的快捷指令
3. **文件系统访问**: 检查照片库目录结构
