## 二、小红书直播方式调研

### 2.1 小红书直播的两种方式

| 方式 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **手机APP开播** | 日常直播、户外直播 | 简单直接、无额外设备 | 需要手机一直在线、耗电 |
| **RTMP推流** | 专业直播、长时间直播 | 可用电脑/服务器、画质可控 | 需要第三方工具、技术门槛 |

### 2.2 小红书RTMP推流

**技术原理：**
```
监控摄像头 → 推流软件（OBS） → RTMP地址 → 小红书直播
```

**获取RTMP地址：**
1. 打开小红书APP → 创作者中心 → 直播
2. 选择「专业模式」或「OBS推流」
3. 获取 RTMP 推流地址 + 推流码（每次开播前需要重新获取）

**推流软件推荐：**
- **OBS Studio**（免费，最通用）
- **Streamlabs OBS**（简化版OBS）
- **vMix**（付费，专业级）

### 2.3 小红书直播规则

⚠️ **重要限制：**
- 不允许无人值守直播（需要人工监管评论区）
- 不允许长时间静音直播（建议有背景音乐）
- 内容需符合平台规范（无违规内容）

## 三、监控摄像头直播技术方案

### 3.1 方案对比

#### 方案 A：监控摄像头 → RTMP 推流器 → 小红书（推荐）

**技术架构：**
```
监控摄像头（RTSP流）
    ↓
中转设备（电脑/服务器）
    ↓
OBS Studio（拉流 + 推流）
    ↓
小红书直播（RTMP）
```

**优点：**
- 灵活性高，可添加字幕、背景音乐
- 支持多摄像头切换
- 可远程操控摄像头（如果支持）

**缺点：**
- 需要一台持续运行的设备
- 技术门槛较高

**成本估算：**
- 中转设备（已有 Mac/Windows 电脑）：¥0
- 监控摄像头（现有）：¥0
- **总成本：¥0** ✅

---

#### 方案 B：带RTMP推流的监控摄像头（免中转）

**技术架构：**
```
监控摄像头（支持RTMP推流）
    ↓
直接推流到小红书RTMP地址
```

**优点：**
- 无需额外中转设备
- 功耗低、体积小

**缺点：**
- 可选摄像头型号少
- RTMP推流地址每次开播前需手动更新（麻烦）
- 无法添加字幕/音乐

**成本估算：**
- 支持RTMP推流的监控摄像头：¥300-800
- **总成本：¥300-800**

---

#### 方案 C：监控摄像头 + 直播推流器（硬件）

**技术架构：**
```
监控摄像头（HDMI/RTSP）
    ↓
直播推流器（硬件编码器）
    ↓
小红书直播（RTMP）
```

**推荐设备：**
- **米家直播摄像头**
- **萤石直播摄像头**
- **RTMP推流器**（如：AV.io HD、Epiphan Pearl）

**优点：**
- 硬件稳定、功耗低
- 可24小时运行
- 支持远程操控

**缺点：**
- 成本较高
- 功能相对固定

**成本估算：**
- 推流器硬件：¥500-2000
- 监控摄像头（现有）：¥0
- **总成本：¥500-2000**

---

### 3.2 最终推荐：方案 A（监控摄像头 → OBS → 小红书）

**理由：**
- 成本最低（利用现有设备）
- 灵活性最高（可后期优化）
- 支持远程操控（如摄像头支持云台）

## 五、技术实现步骤

### 5.1 方案 A 实现步骤

#### Step 1：监控摄像头设置

1. 安装监控摄像头到建军睡觉区域
2. 手机APP连接摄像头WiFi
3. 设置固定IP地址（方便OBS拉流）
4. 测试RTSP流地址（通常格式：`rtsp://用户名:密码@IP地址:端口/路径`）

#### Step 2：OBS 配置

1. **下载安装 OBS Studio**
   - macOS：https://obsproject.com/download
   - Windows：https://obsproject.com/download

2. **添加媒体源**
   - 来源 → + → 媒体源
   - 取消勾选「本地文件」
   - 输入：`rtsp://用户名:密码@摄像头IP:端口/路径`
   - 调整视频/音频解码器

3. **设置场景**
   - 添加文本：建军的睡觉时间 🐱
   - 添加背景音乐（版权音乐，如 lo-fi、白噪音）
   - 设置字幕：欢迎来吸建军~

4. **推流设置**
   - 设置 → 推流
   - 服务：自定义
   - 服务器：从小红书获取的RTMP地址
   - 串流密钥：从小红书获取的推流码
   - 视频码率：2000-4000 Kbps
   - 音频码率：128 Kbps

5. **测试推流**
   - 点击「开始推流」
   - 在小红书APP查看直播画面
   - 调整码率/分辨率确保流畅

#### Step 3：小红书开播

1. 打开小红书APP → 创作者中心 → 直播
2. 选择「专业模式」或「OBS推流」
3. 获取 RTMP 地址 + 推流码
4. 复制到 OBS 设置中
5. 在 OBS 点击「开始推流」
6. 返回小红书APP确认直播开始

#### Step 4：远程操控摄像头

- 打开监控摄像头APP（萤石/米家）
- 远程调整摄像头角度、焦距
- 预设多个点位（全景、特写、侧脸）

### 5.2 自动化方案（可选）

#### 方案 1：定时自动开播

使用脚本定时开启OBS推流：

```bash
#!/bin/bash
# start_live.sh

# 1. 获取小红书RTMP地址（需要手动获取）
RTMP_URL="rtmp://xxxx.xiaohongshu.com/live"
STREAM_KEY="xxxxx"

# 2. 启动OBS并自动推流（需配置OBS自动启动）
# 使用obs-ffmpeg-plugin实现命令行推流
ffmpeg -re -i rtsp://camera_ip/stream \
  -c:v libx264 -preset veryfast -b:v 2500k \
  -c:a aac -b:a 128k \
  -f flv "$RTMP_URL/$STREAM_KEY"
```

#### 方案 2：远程开机+自动开播

使用Wake-on-LAN远程唤醒电脑 + 自动启动OBS：

1. 设置电脑网络唤醒
2. 安装开机启动脚本（自动启动OBS + 推流）

## 七、注意事项

### 7.1 平台规则
- ⚠️ 小红书不允许完全无人值守直播，需有人监管评论区
- ⚠️ 避免长时间静音直播，建议加背景音乐
- ⚠️ 直播内容需符合平台规范，无违规内容

### 7.2 技术风险
- 监控摄像头稳定性（网络波动可能导致掉线）
- 电脑长时间运行稳定性（建议定期重启）
- RTMP推流地址需要每天重新获取（麻烦）

### 7.3 隐私安全
- 监控摄像头账号密码安全
- 不要暴露家庭隐私信息
- 直播时注意背景内容
